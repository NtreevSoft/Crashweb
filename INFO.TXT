
*** 오류 보고를 데이터베이스화 하는 과정

1. 오류 보고는 '클라이언트'가 '공개된 웹 서버'에 리포트를 HTTP POST 방식으로 업로드한다.
2. 그러면, 개발서버에서 보고된 오류 리포트 파일을 긁어 와서 데이터베이스화 한다. (개발서버는 내부용)
3. 이 과정이 완료되면, 개발자는 데이터베이스화된 자료를 웹 브라우저로 열어 보면된다.



*** 각 프로젝트 설명

 - ASPX/Reception - 클라이언트가 보고하는 리포트 파일을 받아서 저장하는 웹 어플리케이션.

 - ASPX/Browser - 개발자가 오류 정보를 조회할 때 사용하는 웹 어플리케이션.

 - SQL - 데이터베이스 테이블 및 스토어드 프로시저 스크립트.

 - CrashGatherer - 리포트 파일을 분석해서 데이터베이스에 밀어 넣는 어플리케이션.


*** 오류를 분석하는 곳(CrashGatherer를 돌리는 곳)에는 Debugging Tools for Windows 설치가 필요합니다.
*** (덤프 파일 분석할 때 CDB를 이용하기 때문입니다)
